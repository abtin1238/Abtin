# آبتین (Abtin Navigator)

مسیریاب هوشمند آفلاین سه‌بعدی — یک اپلیکیشن Flutter برای ناوبری با نقشه‌ی سه‌بعدی، مبتنی بر OpenStreetMap.

---

## ✨ ویژگی‌ها

| ویژگی | وضعیت |
|---|---|
| نقشه سه‌بعدی (MapLibre Native، Vector Tiles) | ✅ |
| GPS Real-Time با هموارسازی موقعیت/جهت | ✅ |
| مسیریابی با دستورات پیچ‌به‌پیچ فارسی | ✅ (فعلاً آنلاین، از OSRM) |
| دانلود نقشه‌ی آفلاین به‌تفکیک استان | ✅ |
| ذخیره مکان‌های مورد علاقه (دیتابیس محلی) | ✅ |
| دیپ‌لینک برای فراخوانی از اپ‌های دیگر (مثل اسنپ) | ✅ |
| دریافت خودکار مجوز و درخواست روشن‌سازی GPS | ✅ |
| انتخاب خودرو (پیکان ساده / مدل سه‌بعدی BMW i8) | ⚠️ رندر سه‌بعدی واقعی هنوز پیاده نشده |
| جستجوی آدرس/POI | ⚠️ فقط UI، به دیتابیس واقعی وصل نیست |

## 🛠 پیش‌نیازها

- Flutter SDK (نسخه پیشنهادی: `3.35.1`، مطابق CI پروژه)
- Dart SDK `>=3.3.0 <4.0.0`
- Android Studio یا فقط Android SDK/NDK (نسخه NDK پیشنهادی: `28.1.13356709`)
- JDK 17

## 🚀 راه‌اندازی

```bash
# نصب وابستگی‌ها
flutter pub get

# ساخت فایل‌های تولیدشده‌ی دیتابیس (Drift) — این مرحله اجباری است
dart run build_runner build --delete-conflicting-outputs

# اجرا روی دستگاه/شبیه‌ساز متصل
flutter run
```

> ⚠️ بدون اجرای مرحله‌ی `build_runner`، پروژه build نمی‌شود چون فایل `lib/core/database/app_database.g.dart` به‌صورت خودکار تولید می‌شود و در ریپو کامیت نشده است.

## 📦 ساخت نسخه‌ی Release

```bash
flutter build apk --release
```
خروجی در مسیر `build/app/outputs/flutter-apk/app-release.apk` قرار می‌گیرد.

> پروژه یک GitHub Action (`.github/workflows/build-from-zip.yml`) دارد که همین مراحل را به‌صورت خودکار روی هر push انجام می‌دهد.

## 🔐 راه‌اندازی امضای ثابت (برای اینکه بیلد جدید، آپدیتِ نصب قبلی باشد نه نصب از نو)

پیش‌تر هر بیلد در گیت‌هاب اکشن با یک کلید **تصادفیِ debug** امضا می‌شد که هر بار در یک ماشین مجازی تازه از نو ساخته می‌شد؛ در نتیجه امضای هر APK با نسخه‌ی قبلی فرق داشت و اندروید اجازه‌ی آپدیت روی نسخه‌ی نصب‌شده را نمی‌داد. برای رفع همیشگی این مشکل:

**۱. یک کیستور ثابت بساز (فقط یک‌بار، روی سیستم خودت):**
```bash
keytool -genkey -v -keystore abtin-release.jks -keyalg RSA -keysize 2048 -validity 10000 -alias abtin
```
از این‌ها یادداشت بردار: پسورد کیستور، الیاس (`abtin`)، پسورد کلید.

**۲. فایل را Base64 کن:**
```bash
base64 -w 0 abtin-release.jks > abtin-release.b64.txt
```

**۳. توی گیت‌هاب ریپو برو به `Settings → Secrets and variables → Actions` و این ۴ Secret رو اضافه کن:**
| نام Secret | مقدار |
|---|---|
| `ABTIN_KEYSTORE_BASE64` | محتوای فایل `abtin-release.b64.txt` |
| `ABTIN_KEYSTORE_PASSWORD` | پسورد کیستور |
| `ABTIN_KEY_ALIAS` | `abtin` (یا هرچی خودت گذاشتی) |
| `ABTIN_KEY_PASSWORD` | پسورد کلید |

بعد از این، هر بیلد بعدی با همین کلید امضا می‌شود و روی گوشی به‌صورت **آپدیت** نصب می‌شود، نه نیاز به Uninstall کردن نسخه‌ی قبلی. کیستور را جایی امن نگه دار — گم‌کردنش یعنی دیگر هیچ‌وقت نمی‌توانی آپدیتی با همان امضا منتشر کنی.

> برای توسعه‌ی محلی (`flutter run` روی سیستم خودت)، بدون این تنظیمات هم مشکلی نیست — به‌صورت خودکار موقتاً از کلید debug استفاده می‌شود.

## 🗂 ساختار پروژه

```
lib/
  core/               # زیرساخت مشترک: دیتابیس، تم، روتر، دیپ‌لینک، مجوزها
  features/
    map/              # صفحه اصلی نقشه و ناوبری
    gps/              # سرویس و providerهای GPS
    routing/          # سرویس مسیریابی (OSRM) و providerهای مسیریابی فعال
    offline_maps/     # دانلود/مدیریت نقشه آفلاین استانی
    saved_places/     # علاقه‌مندی‌ها (دیتابیس محلی)
    search/           # جستجو (UI — نیاز به اتصال به داده واقعی)
    settings/         # صفحه تنظیمات
    voice_settings/   # تنظیمات صدا (UI — نیاز به اتصال به TTS واقعی)
    routes/           # صفحه مسیرها (UI — نیاز به اتصال به تاریخچه واقعی)
    vehicle/          # انتخاب و نمایش خودرو روی نقشه
  shared/widgets/     # ویجت‌های مشترک (GlassPanel, BottomNav, PageHeader)
android/              # پروژه Android Native (Kotlin)
android_manifest_reference/  # مرجع تنظیمات AndroidManifest (برای بازسازی پروژه)
assets/               # فونت‌ها، تصاویر، مدل سه‌بعدی GLB
```

معماری بر پایه‌ی **Feature-based** و مدیریت state با **Riverpod** است.

## ⚠️ محدودیت‌های شناخته‌شده

- **مسیریابی فعلاً آنلاین است** (سرور عمومی OSRM) — برای استفاده‌ی Production، یک سرور مسیریابی آفلاین اختصاصی (پیشنهاد: BRouter) باید جایگزین شود.
- نقشه‌ی آفلاین فعلاً فقط **تایل بصری** دانلود می‌کند، نه دیتای خام POI/گراف جاده — بنابراین جستجو و مسیریابی در حالت کاملاً آفلاین هنوز پشتیبانی نمی‌شوند.
- رندر سه‌بعدی واقعی مدل خودرو (GLB) نیاز به یک لایه Native (مثل Filament) دارد که هنوز اضافه نشده؛ فعلاً یک آیکون دوبعدی جایگزین است.
- برای فهرست کامل کارهای باقی‌مانده و اولویت‌بندی آن‌ها به فایل [`Abtin-Review-TODO.md`](./Abtin-Review-TODO.md) مراجعه کنید.
