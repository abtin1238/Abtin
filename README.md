# آبتین — مسیریاب هوشمند (اسکلت فاز ۱: UI + نقشه آنلاین)

## این پکیج چیست
اسکلت اولیه‌ی اپلیکیشن Flutter شامل:
- تم و ظاهر کامل، دقیقاً برگرفته از پروژه‌ی HTML اصلی (صفحه اصلی سبز، صفحات داخلی آبی-بنفش با گلس‌مورفیسم)
- ۵ صفحه: خانه/نقشه، جستجو، مسیرها، تنظیمات، تنظیمات صدا
- کامپوننت مشترک ناوبری پایین صفحه با همان منطق جابه‌جایی آیتم فعال به وسط (بازنویسی دقیق `js/bottom-nav.js`)
- اتصال نقشه‌ی سه‌بعدی **آنلاین** با `maplibre_gl` (فعلاً با style دمو عمومی MapLibre)
- مسیرهای GoRouter آماده برای گسترش به Deep Link
- Manifest مرجع برای مجوزهای GPS و intent-filter دیپ‌لینک (`abtin://navigate`)

## ⚠️ محدودیت مهم محیطی که این کد در آن تولید شده
این کد در یک sandbox **بدون Flutter SDK نصب‌شده و بدون دسترسی به اینترنت** نوشته شده است.
یعنی:
- کد از نظر نحوی (syntax) با دقت نوشته شده، اما **هرگز `flutter analyze` یا `flutter run` روی آن اجرا نشده.**
- نسخه‌ی دقیق پکیج‌ها (خصوصاً `maplibre_gl`) را باید بعد از `flutter pub get` بررسی و در صورت نیاز به‌روزرسانی کنید — API این پکیج بین نسخه‌ها گاهی تغییر می‌کند.
- پوشه‌های `android/` و `ios/` به‌طور کامل ساخته نشده‌اند (چون فقط `flutter create` با SDK واقعی می‌تواند بولرپلیت درست و به‌روز Gradle/Xcode تولید کند).

## راه‌اندازی روی سیستم شما

```bash
# 1) داخل همین پوشه پروژه:
flutter create . --org ir.abtin --project-name abtin_navigator

# این دستور android/ , ios/ , test/ , .gitignore و غیره را می‌سازد
# بدون این‌که به lib/ و pubspec.yaml موجود آسیب بزند (فایل‌های تکراری را override نمی‌کند مگر تأیید بدهید)

# 2) نصب پکیج‌ها
flutter pub get

# 3) اضافه‌کردن فونت Vazirmatn
# فایل‌های .ttf را از https://github.com/rastikerdar/vazirmatn دانلود و در assets/fonts/ قرار دهید

# 4) ادغام مجوزها و intent-filter دیپ‌لینک
# محتوای android_manifest_reference/AndroidManifest_additions.xml را
# داخل android/app/src/main/AndroidManifest.xml کپی کنید

# 5) اجرا
flutter run
```

## نکته درباره‌ی style نقشه
در `home_screen.dart` فعلاً از style دموی عمومی MapLibre استفاده شده تا فقط رندر نقشه آنلاین تست شود.
برای ظاهر شب‌رنگ (dark) حرفه‌ای مشابه UI فعلی، باید یک Style JSON واقعی (مثلاً از MapTiler، Stadia Maps،
یا سرور Vector Tile اختصاصی خودتان) جایگزین شود — این موضوع را در فاز نقشه‌ی آفلاین دقیق‌تر جمع‌بندی می‌کنیم.

## فاز ۲ — چیزهایی که اضافه شد

- **GPS واقعی**: `LocationPermissionFlow` هنگام باز شدن اپ مجوز موقعیت را می‌گیرد و اگر GPS
  خاموش باشد، دیالوگ سیستمی روشن‌کردن آن را نشان می‌دهد (`core/permissions/location_permission_flow.dart`).
- **هموارسازی GPS**: `LocationService` با یک فیلتر Exponential Moving Average روی موقعیت و Heading
  (`features/gps/data/location_service.dart`) — پرش موقعیت و لرزش چرخش خودرو را کم می‌کند. این یک
  Kalman Filter کامل نیست؛ اگر بخواهید دقیق‌تر شود، در فاز Map Matching (بعد از اتصال روتینگ)
  می‌توان یک Kalman واقعی جایگزین کرد.
- **مارکر خودرو/پیکان**: `VehicleMarker` — دیفالت = پیکان ساده (طبق درخواست شما)، با قابلیت سوییچ به
  حالت «BMW i8» از تنظیمات ← «انتخاب خودرو». **نکته مهم:** چون MapLibre در Flutter لایه‌ی رندر مستقیم
  glTF/GLB روی صحنه‌ی سه‌بعدی ندارد، در حالت BMW i8 فعلاً یک آیکون دوبعدی جایگزین نمایش داده می‌شود؛
  رندر واقعی مدل GLB (`assets/models/bmw_i8.glb`) نیاز به یک لایه‌ی Native (مثلاً Filament روی اندروید)
  دارد که باید در یک فاز جداگانه اضافه شود — این محدودیت واقعی کتابخانه است، نه چیزی که بشود دور زد.
- **مسیریابی با لمس نقشه**: با لمس روی نقشه، یک Pin مقصد و کارت اطلاعات مقصد نمایش داده می‌شود.
  دکمه‌ی «شروع مسیریابی» فعلاً غیرفعال/نشانگر است چون به موتور Valhalla وصل نشده (فاز بعد).
- **دیپ‌لینک واقعی**: `DeepLinkService` با پکیج `app_links` لینک‌های `abtin://navigate?lat=..&lng=..&label=..`
  را می‌گیرد (هم Cold Start هم وقتی اپ باز است) و مستقیماً دوربین را به مقصد می‌برد.
  انتظار می‌رود بعد از merge کردن `AndroidManifest_additions.xml`، اپ‌هایی مثل اسنپ بتوانند این لینک را باز کنند.
- **دیتابیس مکان‌های ذخیره‌شده**: با `drift` (SQLite) — جدول `SavedPlaces` واقعی، بدون هیچ داده‌ی fake.
  صفحه‌ی «علاقه‌مندی‌ها» (`/saved-places`) از تنظیمات در دسترس است.

### ⚠️ یک مرحله‌ی اضافه قبل از اجرا: Code Generation برای drift

جدول‌های `drift` نیاز به یک فایل تولیدی (`app_database.g.dart`) دارند که فقط با `build_runner` (که به
Dart/Flutter SDK واقعی نیاز دارد) ساخته می‌شود — من این فایل را در این sandbox نساختم چون بدون SDK
واقعی، تضمینی برای درستی خروجی‌اش وجود ندارد. بعد از `flutter pub get` این را اجرا کنید:

```bash
dart run build_runner build --delete-conflicting-outputs
```
